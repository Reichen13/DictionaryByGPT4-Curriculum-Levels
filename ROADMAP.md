# Roadmap

## Current Status

这个仓库已经从“课标分级词书”走到了“可用的静态学习 Demo”阶段。

已完成：

- 正式课标分级重排：首页按小学、初中、高中、大学四级、大学六级、更高阶、扩展排序展示
- 独立仓库发布：已迁移到 `Reichen13/DictionaryByGPT4-Curriculum-Levels`
- GitHub Pages 部署：主站和学习页都已可访问
- 学习页 Demo：支持按学段切换 lesson、逐句定位、点词看分级、生词本
- 语音能力：支持预生成句子音频和浏览器 TTS 两种模式
- 字幕舞台：支持句内进度高亮、语速同步、学段快捷入口

当前定位：

- 这不是完整产品，还处在“高可用原型”阶段
- 词书部分已经能长期稳定使用
- 学习页部分已经足够验证产品方向，但 lesson 数据和音频生产链还比较手工

## Product Direction

目标不是做一个单纯的单词列表，而是做一个“按正式课标分级的英语输入学习站”。

核心路径：

1. 从词书分级进入
2. 进入对应学段的句子/视频学习
3. 在句子里点词回看分级和释义
4. 把陌生词加入自己的生词本
5. 逐步扩展到可持续导入内容、可持续生成音频、可持续积累学习记录

## Milestones

### M1 已完成：课标分级词书

- 基于正式课标完成小学、初中、高中的分级映射
- 保留原项目内容和原作者署名
- 形成独立可发布站点

### M2 已完成：最小学习闭环

- 建立 `study.html`
- 支持示例 lesson
- 支持逐句播放、切句、循环、点词和生词本
- 支持更自然一些的浏览器 TTS

### M3 下一阶段：内容生产链

这是最值得优先推进的一段。

目标：

- 不再手工写死 lesson 数据
- 让“视频/字幕/句子/音频”可以批量生成

建议交付项：

- 增加 `lessons/*.json` 规范
- 增加 `scripts/import_lessons.py`
- 支持从 `srt/json/csv` 导入句子时间轴
- 支持批量生成 `study_demo_catalog.json`

完成后收益：

- 新增课程不再需要手改前端
- 以后可以快速扩容小学、初中、高中 lesson

### M4 第二阶段：更像产品

建议交付项：

- 生词本导出为 `json/csv`
- 最近学习记录和断点续学
- lesson 完成度显示
- 学段页与学习页之间的双向跳转优化

完成后收益：

- 用户学习路径更完整
- 页面不再只是 demo，而更像真正可持续使用的学习工具

### M5 第三阶段：更拟人的语音

当前浏览器 TTS 性价比最高，但上限有限。

建议路线：

1. 保留浏览器 TTS 作为零成本默认方案
2. 可选接入真实神经 TTS

优先建议：

- `OpenAI gpt-4o-mini-tts`

原因：

- 音质明显优于浏览器 TTS
- 成本相对可控
- 接入复杂度比自建语音链低

适合的接入方式：

- 后处理批量生成每句音频
- 静态文件部署到仓库或对象存储
- 前端只负责切换音频源，不直接暴露密钥

### M6 第四阶段：真实学习站

建议交付项：

- 后台 lesson 管理
- 用户账户与学习数据同步
- 每个学段的系统化课程目录
- 搜索、收藏、复习、错词回放

这一步才需要认真考虑是否从纯静态站升级到前后端架构。

## Priority Recommendation

如果按“投入产出比”排序，建议接下来这样做：

1. 先做 lesson 导入和内容生产链
2. 再做生词本导出、学习记录、完成度
3. 再决定是否接入神经 TTS
4. 最后再考虑完整后台

## Known Gaps

- 当前只有 3 个示例 lesson
- 非小学/初中/高中学段还没有对应 lesson，只能回跳主词书
- 浏览器 TTS 的实际音色受浏览器和本机环境影响
- 字幕逐词高亮目前基于句内时间估算，不是真实词边界对齐
- 部分原始词条存在重复，这来自原始数据源，不是本仓库新增问题

## Definition Of Done For Next Milestone

下一里程碑建议定义为：

- 可以把一份 `video + subtitles` 或 `sentences + timings` 批量导入成 lesson
- 新 lesson 不需要手改 `study.html`
- 至少补齐 9 个 lesson：小学 3、初中 3、高中 3
- README 里补充 lesson 制作规范
